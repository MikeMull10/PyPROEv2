Design of Experiments

The Design of Experiements tab allows users to either create or import a design matrix that models an optimization problem.

1. Open DOE File

Opens up a prompt for the user to select a Design of Experiements file (.doe).

2. Method

Dropdown for the following options:

  • Factorial
  • Central Composite, Spherical
  • Central Composite, Face-Centered
  • Taguchi Orthogonal Array
  • Latin Hypercube

3. Variable Count

The number of variables in the design. Must be an integer between 1 and 99.

4. Level Count

The number of levels (test points) per variable in the design. Must be an integer between 2 and 20.

5. Data Points (Hypercube Only)

The number of points in the design. Must be an integer between 1 and 99.

6. Function Count

The number of functions in the design. Must be an integer between 0 and 99.

7. Get Design

Generates a popup for the user to input the variable boundaries and the function definitions.

8. Edit Values

Edit the values of the variable ranges (boundaries) and the function definitions. Any changes to the function definitions will automatically be updated and reflected in the design matrix.

9. Add Point

Add a point to the design matrix. Function values will automatically be calculated based on the values of the variables.

10. Clear

Clear the design matrix. This will delete all progress and cannot be undone.

11. Delete Point

Delete a point in the design matrix. The user can only delete points that they added, not the automatically generated ones.

12. Metamodel

Send the design matrix to the metamodeling tab. This button will automatically swap tabs and import the design matrix from the Design of Experiements tab.

13. Compare

Prompts user for input to compare how changing certain variables affects a specific function. There is a minimum requirement of 2 variables and 1 function for this feature to work. This feature is only visible after the design matrix has been created.

14. Design Matrix Table

The table where the points of the design matrix are displayed. The normalized variable values (i.e. X1 (normalized)) are not editable nor do they directly affect any results. The real variables and functions are directly editable. If the function(s) have a defined equation, then updating the variable value(s) will automically update the function value(s).

Metamodeling

The Metamodeling tab allows users to take their design matrix and generate equations to help solve the optimization problem.

1. Open DOE File

Opens up a prompt for the user to select a Design of Experiements file (.doe).

2. Method

Dropdown for the following options:

  • Polynomial Regression
  • Radial Basis Functions

3. Function

Determines the type of the method used to generate the functions.
For Polynomial Regression, there are the following options:

  • Linear Polynomial
  • Quadratic Polynomial with no Interaction
  • Quadratic Polynomial with Interaction

For Radial Basis Functions, there are the following options:

  • Linear
  • Cubic
  • Thin Plate Spline
  • Gaussian
  • Multiquadratic
  • Inverse Multiquadratic
  • Compactly Supported (2,0)
  • Compactly Supported (2,1)
  • Compactly Supported (2,2)
  • Compactly Supported (3,0)
  • Compactly Supported (3,1)
  • Compactly Supported (3,2)
  • Compactly Supported (3,3)

4. Polynomial Order (Radial Basis Functions Only)

Dropdown for the following options:

  • 0 - The RBF approximation only considers the radial basis function values to interpolate or approximate the data.
  • 1 - The interpolation can capture linear trends in the data, in addition to the localized effects modeled by the RBF.

5. Evaluate

Prompt the user for comma separated values for each variable in the design. This will output the value of each variable entered and the calculated value of each function.

6. Generate

Generates the functions based on the previous settings (either Polynomial Regression or Radial Basis Functions).

7. Clear

Resets the data. The DOE file does not need to be re-imported, but everything else is deleted. This cannot be undone.

8. Formulate

Send the generated functions to the Formulation tab.

Formulation

The Formulation tab allows users to use their mathematical equations and other design variables to create a Function file (.fnc) to perform optimization.

1. Load FNC File

Opens up a prompt for the user to select a Function file (.fnc).

2. Optimization Functions

Dropdown for the user to select the function(s) they wish to optimize.

3. Variable

Dropdown for the user to select the variable(s) needed for optimization.

4. Equality Constraints

Dropdown for the user to select the function(s) that are the equality constraints.

5. Inequality Constraints

Dropdown for the user to select the function(s) that are the inequality constraints.

6. Constraint Adjustments

Prompts the user to enter offsets for the constraint equations. For example, if the constraint function is F2 and the desired constraint is F2 = 3, the value 3 can be input here as an offset instead of being directly incorporated into the equation itself.

7. Gradient Policy

Generates a popup to remind users of how PyPROE handles gradients. It has no functional use.

8. Reset Environment

Resets the data. This cannot be undone.

9. Function and Variable Dropdowns

Dropdowns for the different functions and variables. Metamodel Functions are functions imported directly from the Metamodeling tab. Imported Functions are functions imported from a Function file (.fnc). User Defined Functions are functions that were created in the Formulation tab directly. Variable ranges are imported from either the Metamodeling tab or the FNC file.

10. Preview

Previews the selected function or variable. They cannot be edited directly in this section.

11. Create New Function

Space for manually defining new functions or variables. Functions should be defined starting with F and variables should be defined starting with X. These will be displayed under User Defined Functions (UDF) for functions and Variable Ranges for variables.

Optimization

The Optimization tab allows users to find the solution to their optimization problems using a variety of solving methods.

1. Open FNC File

Opens up a prompt for the user to select a Function file (.fnc).

2. OPT Solver

Dropdown for the following options:

1. SLSQP (Single Obj)

Single-objective optimization method that minimizes a single objective function. It handles both equality and inequality constraints and iteratively adjusts decision variables to find the optimal solution.

Uses the following parameters:

  • Grid Size

2. SLSQP (Weighted Sum Formulation)

Multi-objective optimization method that uses the Weighted Sum Formulation (WSF) to combine multiple objective functions into a single scalar value. The method generates a Pareto front, a set of solutions where no point is strictly better than another with respect to all objectives. Requires at least two objectives and iterates over different weightings to approximate the trade-offs between them.

Uses the following parameters:

  • Normalize function?
  • Grid Size
  • Min-Weight
  • Weight Increment

3. NSGA-II

Non-dominated sorting genetic algorithm with reference ponts (NSGA-III). Replaces NSGA-II's crowding distance-based selection with one using reference points. At a high level, the advantage of reference points is two-fold. First, they can provide a better distribution of points, especially on problems with many objectives where most solutions tend to be non-dominated. Second, the reference points can be selected to target specific regions of interest.

Uses the following parameters:

  • Iterations/Generations
  • Population Size
  • Crossover Rate
  • Mutation Rate

4. NSGA-III

NSGA-III (Non-dominated Sorting Genetic Algorithm III) is an extension of NSGA-II, designed to address many-objective optimization problems. It handles problems with more than three conflicting objectives and uses a reference-point-based approach to preserve diversity along the Pareto front. Unlike NSGA-II, which struggles with many objectives, NSGA-III ensures a well-distributed set of solutions even in high-dimensional objective spaces.

Uses the following parameters:

  • Iterations/Generations
  • Crossover Rate
  • Mutation Rate
  • Outer Divisions
  • Inner Divisions

5. EpsMOA

EpsMOA (Epsilon-based Multi-objective Optimization Algorithm) is an evolutionary algorithm designed to handle multi-objective optimization problems. Unlike traditional dominance-based methods, EpsMOA uses the epsilon indicator to evaluate solutions and guide the population toward a well-converged and diverse approximation of the Pareto front. The algorithm maintains an archive of the best-found solutions and updates it iteratively, ensuring that the solutions remain well-distributed along the Pareto front.

Uses the following parameters:

  • Epsilons
  • Iterations/Generations
  • Population Size
  • Crossover Rate
  • Mutation Rate

3. Optimization Settings

1. Grid Size

Integer between 1 and 100. Default: 5.

Determines the number of equally spaced samples generated along each dimension in the grid. A higher grid size increases the number of guesses by creating a finer grid, while a lower grid size reduces the number of guesses by creating a coarser grid.

2. Iterations/Generations

Integer between 1 and 10000. Default: 100.

Defines the number of cycles the algorithm will run.

3. Min-Weight

Decimal between 0 and 1 exclusively. Default: 0.01.

Defines the lower bound for weights in weighted formulations.

4. Weight Increment

Decimal between 0 and 1 exclusively. Default: 0.01.

Defines the adjustment interval for weights.

5. Epsilons

List of decimals. Each decimal should be between 0 and 1 exclusively. Default: 0.005.

The step size for adjusting epsilon values during optimization. Smaller increments allow for finer exploration, while larger increments expedite convergence.

6. Population Size

Integer between 1 and 100000. Default: 100.

Defines the number of candidate solutions in the optimization process.

7. Mutation Rate

Integer (percentage) between 1 and 100. Default: 1%.

Controls the probability of recombination.

8. Crossover Rate

Integer (percentage) between 1 and 100. Default: 90%.

Defines the frequency of random genetic changes.

9. Outer Divisions

Integer between 0 and 100000. Default: 100.

Defines the primary level of segmentation.

10. Inner Divisions

Integer between 0 and 100000. Default: 25.

Refines the divisions created by outer segmentation.

11. Normalize function?

Boolean Yes/No. Default: Yes.

Determines whether functions and gradients will be normalized before optimization.

4. Start

Begins the calculating the solution to the optimization problem.

5. Stop

Cancels calculating the solution to the optimization problem.

6. Clear

Clear the solution. This cannot be undone.

7. View Graph

Open a separate window that displays the graphed solution.

Opens up a prompt for the user to select a Function file (.fnc).

2. Generate Contour

Creates a graph of the countor plot for the function.

Gradients

The Gradients tab allows users to generate the gradients for equations.

1. Open FNC File

Opens up a prompt for the user to select a Function file (.fnc).

2. Generate Blank Template

Creates a blank template for users to manually enter information about the optimization problem.

3. Generate Gradients

Uses the FNC file data (whether imported from a file or manually created) and calculates the gradients. NOTE: This will not work of any functions use EXP or ABS as these mathematical functions make calculating gradients impossible currently.

Design of Experiment (DOE) Files

1. Introduction

DOE files are used to define the factorial design of an optimization problem including the data points, variables, design levels, and functions. This section describes the syntax and structure of a DOE file.

2. Comments

Any line starting with # will be considered a comment and ignored by the software.

3. Required Variables

The number of points, variables, levels, and functions all must be defined. This information must be in the first line of the file (not including comment lines) and be separated by spaces. The number of spaces is irrelevant. For example:

# NUM_POINTS NUM_VARS NUM_LEVELS NUM_FUNCS
27 3 3 2

Function (FNC) Files

1. Introduction

FNC files are used to define optimization problems, including the variables, objective functions, and constraints. This section describes the syntax and structure of an FNC file.

2. Comments

Any line starting with # will be considered a comment and ignored by the software.

3. Variable Definitions

Variables must be declared in the following way:

VARIABLE: MIN, MAX, *TYPE, *INCREMENT

For example:

X1: 3.0, 7.0, REAL, 0.000001

TYPE and INCREMENT are optional, and as of the current version of PyPROE, they do not affect the program in any way. The variable's name is crucial to its functionality. Variables must be named in the following sequence:

X1, X2, X3 ...

4. Objective Functions & Constraints

Objective functions should be defined as in the example below:

O1 = F1;

Objectives should be declared sequentially, starting with O1.

Equality and inequality constraints should be defined as in the examples below:

EC1 = F1;
INEC1 = F1;

IMPORTANT NOTE: be sure to include the semicolon ; after the end of each declaration.

5. Function Definitions

Function definitions are similar to the objective functions and constraints. For example:

F1 = X1^2;

Again, please make sure there is a semicolon at the end of the function definition. This format allows for multiple lines, so if there is no semicolon at the end, the program will look to add the next line to the function. Also, make sure your equation is defined explicitly. For example, 5x will cause an error. Make sure that you define it as: 5 * x.

The following special functions are supported in PyPROE:

  • ABS
  • POW
  • LOG
  • LN
  • SIN / ASIN / SINH
  • COS / ACOS / COSH
  • TAN / ATAN / TANH
  • SQRT
  • CEIL
  • FLOOR
  • EXP

6. Gradients

To define a gradient function in PyPROE, you need to use the following format:

G + FUNCTION_NAME + either '-' or '_' + VARIABLE_NAME = ...;

The FUNCTION_NAME is the name of the function (e.g., F1), and the VARIABLE_NAME is the variable with respect to which the gradient is being calculated (e.g., X1).

Complete example:

GF1_X1 = ...;
GF1-X1 = ...;

7. Troubleshooting

  • Make sure there is a semi-colon ; after each definition.
  • Make sure that any multiplication is explicitly defined. I.e., 5x is defined as 5 * x.
  • Make sure that any lines meant to be comments start with #.

8. Full Example

Here is a complete example of an FNC file:

# Example.fnc *VARIABLE X1: 0, 5, REAL, 0.000001 X2: 0, 3, REAL, 0.000001 *OBJECTIVE O1 = F1; O2 = F2; *EQUALITY-CONSTRAINT *INEQUALITY-CONSTRAINT INEC1 = F3; INEC2 = F4; *FUNCTION F1 = 4 * X1^2 + 4 * X2^2; F2 = (X1 - 5)^2 + (X2 - 5)^2; F3 = (X1 - 5)^2 + X2^2 - 25; F4 = -(X1 - 8)^2 - (X2 + 3)^2 + 7.7; *GRADIENT GF1_X1 = 8*X1; GF1_X2 = 8*X2; GF2_X1 = 2*X1 - 10; GF2_X2 = 2*X2 - 10; GF3_X1 = 2*X1 - 10; GF3_X2 = 2*X2; GF4_X1 = 16 - 2*X1; GF4_X2 = -2*X2 - 6;
